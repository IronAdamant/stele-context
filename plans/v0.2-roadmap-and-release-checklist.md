# ChunkForge v0.2 Roadmap & Git Release Checklist

## Executive Summary

This document outlines the plan to address gaps identified in v0.1.0 and prepare for a proper open-source release. The work is organized into three phases:

1. **Phase 1: Release Preparation** (Immediate) - Get v0.1.0 ready for public consumption
2. **Phase 2: Core Improvements** (v0.1.x) - Address critical gaps
3. **Phase 3: Feature Expansion** (v0.2.0) - Major new capabilities

## Phase 3: Feature Expansion (v0.2.0)

### 3.1 Vector Index for Fast Similarity Search

**Priority: 🟡 Important**

**Current Problem:** O(n) scan to find similar chunks

**Solution Options:**
1. **Pure Python HNSW** - No dependencies, slower
2. **FAISS** - Fast, but adds dependency
3. **SQLite FTS5** - Already have SQLite, limited to text search

**Recommendation:** Implement pure Python HNSW with optional FAISS acceleration

**Implementation:**
```python
# chunkforge/index.py
class VectorIndex:
    """HNSW-based vector index for fast similarity search."""
    
    def __init__(self, dim: int = 128, M: int = 16, ef: int = 200):
        self.dim = dim
        self.M = M  # Max connections per node
        self.ef = ef  # Search width
        
    def add(self, chunk_id: str, vector: np.ndarray):
        """Add vector to index."""
        
    def search(self, query: np.ndarray, k: int = 10) -> List[Tuple[str, float]]:
        """Find k nearest neighbors."""
```

- Python (ast module - stdlib)

- JSON/YAML (structure-based)

### 3.3 Compression for KV-Cache

**Priority: 🟢 Nice-to-have**

**Current Problem:** KV-cache files can be large

**Solution:** Add zstd compression

**Implementation:**
```python
# In storage.py
import zstandard as zstd

def store_kv_state(...):
    if self.compression:
        compressed = self.compressor.compress(serialized)
        kv_path.write_bytes(compressed)
```

**Trade-off:** Adds dependency (zstandard), but significant space savings (50-80%)

### 3.4 Chunk Versioning & History

**Priority: 🟢 Nice-to-have**

**Feature:** Track chunk changes over time

**Use Case:** "Show me what changed in this file over the last week"

**Implementation:**
```sql
CREATE TABLE chunk_history (
    chunk_id TEXT,
    version INTEGER,
    content_hash TEXT,
    semantic_signature BLOB,
    created_at REAL,
    PRIMARY KEY (chunk_id, version)
);
```

### 3.5 Advanced Semantic Signatures

**Priority: 🟢 Nice-to-have**

**Current:** TF-style features (trigrams, word freq)

**Improvements:**
1. **TF-IDF weighting** - Better term importance
2. **Local sentence embeddings** - Use small ONNX model
3. **AST features for code** - Function signatures, imports
4. **Structural features** - Heading hierarchy, list depth

**Constraint:** Must remain offline and CPU-only

### 3.6 Multi-Document Sessions

**Priority: 🟡 Important**

**Current:** Sessions are independent

**Enhancement:** Link related documents in a session

**Use Case:** "This session includes: main.py, utils.py, config.py"

**Implementation:**
```python
cf.create_session("coding-session", documents=["main.py", "utils.py"])
```

### 3.7 Selective KV Loading

**Priority: 🟡 Important**

**Current:** Load all chunks or none

**Enhancement:** Load only chunks matching a query

**Use Case:** "Only load KV for chunks related to authentication"

**Implementation:**
```python
cf.load_session_kv(
    session_id="my-session",
    query="authentication",
    top_k=20
)
```
